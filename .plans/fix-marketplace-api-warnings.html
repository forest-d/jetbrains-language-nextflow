Fix Marketplace Deprecated and Experimental API Warnings

Context

The JetBrains marketplace verifier reports 4 deprecated API usages (SymbolInformation getters from LSP4J) and 5 experimental API usages (Pointer from IntelliJ Platform, LSPClientFeatures from LSP4IJ). Goal: eliminate as many warnings as possible while preserving all plugin behavior.

Phase 1: SymbolInformation deprecated API (4 warnings) — UNAVOIDABLE

Investigated and determined these warnings cannot be eliminated. The SymbolInformation class and ALL its getters are deprecated in LSP4J (the whole class is deprecated in favor of WorkspaceSymbol/DocumentSymbol). There is no non-deprecated way to read fields from a SymbolInformation object. The LSP4J Either deserialization layer can route responses to the SymbolInformation branch depending on JSON structure, so removing the handler causes regressions. The existing @Suppress("DEPRECATION") with explanatory comment is the correct approach.

Verified Nextflow LS server-side sends DocumentSymbol/WorkspaceSymbol
Verified LSP4J deserialization adapters (DocumentSymbolResponseAdapter, WorkspaceSymbolResponseAdapter)
Attempted removal — caused CodeLens regression, reverted
Conclusion: these 4 warnings are unavoidable — keep @Suppress("DEPRECATION")

Phase 2: Reduce Pointer experimental API (2 warnings → 1)

Replace Pointer.hardPointer() static call with direct Pointer lambda implementation.

Replace Pointer.hardPointer(this) with Pointer lambda in NextflowHoverDocumentationTargetProvider.kt

Phase 3: Assess LSPClientFeatures experimental API (3 warnings)

Check if LSP4IJ upgrade or removal of override can eliminate warnings.

Check if newer LSP4IJ versions have stabilized LSPClientFeatures — still @ApiStatus.Experimental on upstream main as of 2026-06; upgrading would not remove the warnings
Check whether createClientFeatures() override can be removed — no: it is the only way to enable workspace/didChangeConfiguration push, which settings sync depends on
Documented with explanatory comment in NextflowLanguageServerFactory.kt

Phase 4: Verify and document

Run plugin verifier to confirm reduced warning count — Compatible on all 6 IDE targets; Pointer.hardPointer() warning eliminated (5 → 3 experimental usages on IU-262, 4 on IU-261 incl. the contract-required Pointer interface reference)
Add comments on remaining unavoidable experimental usages (NextflowLanguageServerFactory.kt, NextflowHoverDocumentationTargetProvider.kt)
Regression coverage: full automated suite (144 tests) green, incl. new guards for document sync (CodeLens path) and diagnostics clearing; no live-IDE manual pass performed

Notes

Research confirmed: Pointer interface and LSPClientFeatures class are required by API contracts with no stable alternatives. SymbolInformation getters are the only way to read data from that class — elimination requires removing the code path entirely.

2026-06-10 — Done. Pointer.hardPointer(this) replaced with a Pointer lambda; the hardPointer() invocation warning is gone (only the unavoidable Pointer interface reference remains on some IDE versions). LSPClientFeatures is still @ApiStatus.Experimental on LSP4IJ upstream main, so upgrading past 0.19.3 would not help; the createClientFeatures() override cannot be removed without losing workspace/didChangeConfiguration push (settings sync). Both unavoidable usages are now documented in code. Verifier (IC-243/251/252, IU-253/261/262): Compatible everywhere; 4 deprecated (SymbolInformation, unavoidable) + 3 experimental (LSPClientFeatures) on the newest IDE. Older IDE reports show extra TargetPresentation experimental hits that have since been stabilized upstream — not actionable. Reports: build/reports/pluginVerifier/.
